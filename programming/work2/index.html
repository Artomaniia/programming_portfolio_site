    Описание задания (Calculate)¶

    Тема¶

    Реализация функции деления с заданной точностью и загрузкой параметров из конфигурационного файла.

    Ссылка на репозиторий¶

    Репозиторий с выполненной работой:
    https://gitverse.ru/Artomaniia/herzen_lessons_lello_poo/content/master/2_course/calculate/main.py

    Формулировка задания¶

    Необходимо было выполнить три части работы:

    1. Написать функцию calculate, которая принимает два операнда и выполняет деление первого операнда на второй.
    2. Добавить ключевой аргумент epsilon, отвечающий за точность вычисления. Значение по умолчанию — 0.0001.
    3. Написать функцию load_params, которая считывает значение epsilon из конфигурационного файла settings.ini.
    4. Подготовить тесты для проверки работы функций calculate и load_params.

    Что было сделано¶

    В репозитории реализована программа на Python, состоящая из трёх основных файлов:

    • main.py — основной файл с реализацией функций;
    • settings.ini — конфигурационный файл, из которого считывается значение точности;
    • test_main.py — файл с тестами для проверки работы программы.

    Реализация функции calculate¶

    Функция calculate принимает два числа:

    calculate(a, b, *, epsilon=0.0001)

    где:

    • a — первый операнд;
    • b — второй операнд;
    • epsilon — точность округления результата.

    Функция выполняет деление a / b и возвращает результат, округлённый в зависимости от переданного значения epsilon.

    Также в функции реализованы проверки:

    • если второй операнд равен нулю, вызывается исключение ZeroDivisionError;
    • если epsilon выходит за допустимый диапазон, вызывается исключение ValueError;
    • значение epsilon округляется до безопасного количества знаков после запятой.

    Реализация функции load_params¶

    Функция load_params предназначена для чтения значения точности из конфигурационного файла settings.ini.

    Пример содержимого файла:

    [params]
epsilon = 0.0000001

    Функция считывает параметр epsilon, преобразует его в число и проверяет, входит ли он в допустимый диапазон.

    Если значение отсутствует, записано в неправильном формате или находится вне допустимого диапазона, используется значение по умолчанию:

    0.0001

    Такой подход позволяет программе продолжать работу даже при ошибках в конфигурационном файле.

    Тестирование¶

    Для проверки работы программы был создан файл test_main.py.

    В нём проверяется работа функции calculate на следующих случаях:

    calculate(1, 2, epsilon=0.1)      # ожидаемый результат: 0.5
calculate(1, 1000, epsilon=0.001) # ожидаемый результат: 0.001
calculate(1, 0)                   # должно возникнуть исключение ZeroDivisionError

    Также проверяется работа функции load_params:

    • чтение значения epsilon из файла settings.ini;
    • обработка ситуации, когда файл отсутствует;
    • обработка значения epsilon, которое выходит за допустимый диапазон;
    • обработка ситуации, когда вместо числа в конфигурационном файле записано некорректное значение.

    Как запустить программу¶

    Для запуска основного файла можно использовать команду:

    python main.py

    Как запустить тесты¶

    Для запуска тестов можно выполнить команду:

    python test_main.py

    Примечание¶

    В задании диапазон точности указан как 10**-1 < epsilon < 10**-9, однако в коде используется логически корректная запись:

    1e-9 < epsilon < 1e-1

    То есть значение epsilon должно быть больше 0.000000001 и меньше 0.1.

    Также в реализации load_params предусмотрена защита от ошибок конфигурационного файла: если параметр невозможно считать или он некорректен, программа использует стандартное значение точности 0.0001.

    Итог¶

    В результате была реализована программа, которая выполняет деление двух чисел с заданной точностью, умеет загружать параметр точности из конфигурационного файла и содержит тесты для проверки основных сценариев работы, включая деление на ноль и ошибки в settings.ini.
