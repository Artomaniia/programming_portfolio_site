## Тема работы

## Основные моменты доклада

Паттерн **Decorator** позволяет добавлять объекту новое поведение без изменения его исходного класса. Для этого объект помещается внутрь другого объекта — декоратора, который имеет тот же интерфейс и дополняет работу исходного объекта.

Главная идея паттерна заключается в том, что поведение можно расширять динамически, комбинируя несколько декораторов между собой. Это помогает избежать создания большого количества классов-наследников для каждой возможной комбинации поведения.

В докладе также рассматривалось, что в Python слово «декоратор» может использоваться в двух смыслах:

1. как классический паттерн проектирования Decorator;
2. как языковой механизм Python с использованием синтаксиса `@decorator`.

Оба подхода связаны общей идеей: добавить новую логику, не изменяя исходный объект или функцию напрямую.

## Примеры из репозитория

В репозитории представлены примеры, демонстрирующие работу паттерна:

- пример с напитками, где к базовому объекту добавляются дополнительные компоненты, например молоко, сироп или тоник;
- пример простого функционального декоратора, который выполняет действия до и после вызова функции.

Эти примеры показывают, как с помощью обёрток можно расширять поведение программы без переписывания уже существующего кода.
