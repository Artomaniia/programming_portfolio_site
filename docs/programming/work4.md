# Описание выполненной части проекта

## Проект

**Telegram-бот для библиотеки** — командный учебный проект, посвящённый разработке бота для взаимодействия пользователя с библиотечной системой.

Бот предназначен для того, чтобы пользователь мог пройти регистрацию, посмотреть список библиотек, найти книги в каталоге и забронировать нужную книгу.

## Ссылка на репозиторий

Репозиторий с проектом:  
https://github.com/svgab/LibraryBot/tree/main

## Формат работы

Проект выполнялся в команде, поэтому работа была разделена между несколькими участниками.  
В данном описании указана **моя личная зона ответственности** в рамках проекта.

## Моя зона ответственности

В рамках проекта я занимался следующими частями:

1. разработкой модели базы данных через **Pony ORM**;
2. проектированием структуры хранения данных для библиотеки, книг, пользователей и бронирований;
3. разработкой схемы работы Telegram-бота и логики пользовательского сценария.

---

# 1. Разработка модели базы данных через Pony ORM

Одной из основных задач было продумать структуру базы данных, которая могла бы использоваться Telegram-ботом для хранения информации о пользователях, книгах, библиотеках и бронированиях.

Для работы с базой данных была выбрана библиотека **Pony ORM**, позволяющая описывать таблицы базы данных в виде Python-классов и удобно работать со связями между сущностями.

## Основные сущности базы данных

В модели базы данных были предусмотрены следующие сущности:

| Сущность | Назначение |
|---|---|
| `Book` | Хранение информации о книгах |
| `User` | Хранение данных пользователей Telegram-бота |
| `UserBook` | Хранение информации о бронировании книг пользователями |
| `Library` | Хранение информации о библиотеках |
| `LibraryBook` | Связь между библиотеками и книгами |
| `LibraryHours` | Хранение стандартного расписания работы библиотек |
| `LibraryExceptions` | Хранение разовых исключений в расписании |
| `LibraryRecurringExceptions` | Хранение повторяющихся исключений в расписании |

## Сущность `Book`

Сущность `Book` предназначена для хранения информации о книгах в каталоге.

В неё входят такие данные, как:

- ISBN;
- автор;
- название;
- издательство;
- год издания;
- язык;
- теги;
- количество страниц.

Также у книги предусмотрены связи с бронированиями и библиотеками, в которых она доступна.

## Сущность `User`

Сущность `User` используется для хранения информации о пользователях Telegram-бота.

В неё входят:

- имя пользователя;
- фамилия;
- номер телефона;
- email.

Эти данные необходимы для дальнейшего бронирования книг и связи пользователя с его заказами.

## Сущность `UserBook`

Сущность `UserBook` используется для хранения информации о книгах, которые были забронированы пользователем.

Она связывает пользователя и книгу, а также хранит дополнительные данные:

- статус бронирования;
- дату окончания бронирования;
- возможность продления или дополнительное состояние бронирования.

Такая сущность нужна для того, чтобы бот мог отслеживать, какие книги находятся у пользователя, активна ли бронь и когда она заканчивается.

## Сущность `Library`

Сущность `Library` отвечает за хранение информации о библиотеках.

В неё входят:

- идентификатор библиотеки;
- название;
- адрес;
- номер телефона;
- дополнительный идентификатор или ссылка, например `vk_id`.

Эта сущность нужна для вывода списка библиотек пользователю и отображения информации о выбранной библиотеке.

## Сущность `LibraryBook`

Сущность `LibraryBook` связывает книги и библиотеки.

Она позволяет хранить информацию о том, какие книги есть в конкретной библиотеке и в каком количестве.  
Таким образом, одна и та же книга может находиться в разных библиотеках, а одна библиотека может содержать множество книг.

## Сущности расписания

Для работы с расписанием библиотек были предусмотрены отдельные сущности:

- `LibraryHours` — стандартные часы работы по дням недели;
- `LibraryExceptions` — разовые изменения расписания, например закрытие библиотеки в конкретную дату;
- `LibraryRecurringExceptions` — повторяющиеся исключения, например регулярные изменения в определённые дни недели.

Такое разделение позволяет хранить не только обычное расписание, но и нестандартные ситуации: выходные дни, сокращённые часы работы, временное закрытие библиотеки и другие исключения.

# 2. Связи между сущностями

При проектировании базы данных были продуманы связи между основными объектами системы.

## Пользователь и бронирование

Пользователь может иметь связанные с ним записи о бронировании книг.  
Для этого используется сущность `UserBook`, которая хранит информацию о конкретной брони.

## Книга и библиотека

Книга может находиться в нескольких библиотеках, а библиотека может содержать множество книг.  
Для такой связи используется промежуточная сущность `LibraryBook`.

Это позволяет не просто связать книгу и библиотеку, но и хранить дополнительные данные, например количество экземпляров книги в конкретной библиотеке.

## Библиотека и расписание

Для библиотеки предусмотрено хранение расписания работы и исключений из него.  
Это нужно для корректного отображения пользователю информации о том, когда библиотека открыта.

# 3. Разработка схемы работы Telegram-бота

Помимо модели базы данных, я занимался проектированием логики работы Telegram-бота.

Была составлена схема пользовательского сценария: от первого запуска бота до поиска и бронирования книг.

## Основной сценарий работы

Пользовательский сценарий начинается с команды запуска бота.

После запуска бот приветствует пользователя и сообщает, что для дальнейшей работы необходима авторизация.

## Регистрация пользователя

Для регистрации бот последовательно запрашивает у пользователя:

1. имя;
2. фамилию;
3. номер телефона;
4. email.

После ввода этих данных регистрация считается завершённой, а информация пользователя может быть сохранена в базе данных.

## Главное меню

После регистрации пользователь попадает в главное меню.

В нём предусмотрены основные действия:

- просмотр списка библиотек;
- поиск книг в каталоге.

## Просмотр списка библиотек

При выборе пункта со списком библиотек бот должен выводить пользователю информацию о доступных библиотеках.

В вывод могут входить:

- название библиотеки;
- адрес;
- телефон;
- часы работы.

Для этого используются данные из сущностей `Library`, `LibraryHours` и сущностей с исключениями расписания.

## Поиск книг

При выборе поиска книг пользователь может искать нужную книгу в каталоге.

Поиск может быть связан с данными, которые хранятся в сущности `Book`, например:

- названием книги;
- автором;
- тегами;
- годом издания;
- языком.

После нахождения книги бот может показать информацию о ней и наличие в библиотеках.

## Бронирование книги

После выбора книги пользователь может оформить бронирование.

Для этого в базе данных создаётся или обновляется запись, связывающая пользователя и выбранную книгу.  
За это отвечает сущность `UserBook`.

В записи можно хранить:

- пользователя;
- выбранную книгу;
- статус бронирования;
- дату окончания брони.

# 4. Результат выполненной работы

В рамках моей части проекта была подготовлена логическая основа для работы Telegram-бота:

- разработана структура базы данных;
- выделены основные сущности предметной области;
- продуманы связи между пользователями, книгами, библиотеками и бронированиями;
- добавлена возможность учитывать расписание библиотек;
- отдельно продуманы исключения из расписания;
- составлена схема пользовательского взаимодействия с ботом.

Эта часть работы является важной основой проекта, потому что именно модель данных определяет, как бот будет хранить информацию, обрабатывать действия пользователя и связывать между собой книги, библиотеки и бронирования.
