## Тема работы

В рамках лабораторной работы была реализована программа на Python для получения курсов валют Центрального банка РФ в объектно-ориентированном стиле.

Основной акцент работы — применение шаблона проектирования **Singleton** / **«Одиночка»**. Этот шаблон нужен для того, чтобы у класса существовал только один экземпляр, даже если объект создаётся несколько раз.

## Что требовалось сделать

По заданию необходимо было:

- реализовать получение курсов валют с сайта ЦБ РФ;
- оформить решение в объектно-ориентированном стиле;
- реализовать шаблон **«Одиночка»** через метакласс;
- хранить значения валюты не одним числом с плавающей точкой, а отдельно как целую и дробную часть;
- учитывать номинал валюты, если он отличается от `1`;
- добавить ограничение на слишком частые запросы;
- реализовать построение графика курсов валют и сохранить его в файл `currencies.jpg`;
- написать тесты;
- добавить документацию и аннотации типов.

## Что было реализовано

В проекте реализован класс `CurrenciesRates`, отвечающий за работу с курсами валют.

Класс выполняет следующие задачи:

- получает данные о валютах;
- принимает список идентификаторов валют, например `R01035`, `R01335`, `R01700J`;
- возвращает результат в виде списка словарей;
- хранит курс валюты в формате `("целая часть", "дробная часть")`;
- сохраняет номинал валюты, если он не равен `1`;
- обрабатывает неправильные коды валют;
- хранит результат последнего запроса;
- контролирует минимальный интервал между запросами;
- строит график курсов валют и сохраняет его в файл `currencies.jpg`.

## Реализация шаблона Singleton

Для реализации шаблона **Singleton** был создан метакласс `SingletonMeta`.

Идея работы следующая:

```python
first = CurrenciesRates(["R01035"])
second = CurrenciesRates(["R01335"])

print(first is second)  # True
```

Даже если объект класса `CurrenciesRates` создаётся несколько раз, фактически используется один и тот же экземпляр.

Это удобно для данной задачи, потому что сервис получения валют должен иметь единое состояние:

- список выбранных валют;
- результат последнего запроса;
- время последнего обращения;
- настройки интервала между запросами.

## Пример результата

Пример корректного результата:

```python
[
    {'GBP': ('Фунт стерлингов Соединенного королевства', ('113', '2069'))},
    {'KZT': ('Казахстанских тенге', ('19', '8264'), 100)},
    {'TRY': ('Турецких лир', ('33', '1224'), 10)}
]
```

Если передан неправильный идентификатор валюты, возвращается словарь с этим кодом и значением `None`:

```python
[{'R9999': None}]
```

## Что проверяется в тестах

В тестах проверяется:

- что класс `CurrenciesRates` действительно работает как Singleton;
- что при создании нескольких объектов возвращается один и тот же экземпляр;
- что неправильный код валюты возвращает результат вида `{'R9999': None}`;
- что корректные коды валют обрабатываются правильно;
- что значение курса разделяется на целую и дробную часть;
- что график сохраняется в файл.
