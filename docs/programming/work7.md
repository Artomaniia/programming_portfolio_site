## Тема работы

В рамках лабораторной работы был изучен и реализован структурный паттерн проектирования **Decorator**.

Паттерн «Декоратор» позволяет добавлять объекту новое поведение без изменения его исходного класса. В данной работе это показано на простом примере получения курсов валют и преобразования результата в разные форматы.

## Суть задания

Необходимо было разработать программу, в которой есть базовый компонент для получения данных о курсах валют, а также дополнительные декораторы, расширяющие его поведение.

Базовый компонент возвращает данные в формате **JSON**, а декораторы позволяют получить эти же данные в других форматах:

- **YAML**;
- **CSV**.

Также требовалось реализовать сохранение результата в файлы, использовать абстрактные классы, добавить документацию, аннотации типов и тесты.

## Что было сделано

В проекте реализован общий интерфейс `CurrencyRatesComponent`, от которого наследуются базовый компонент и декораторы.

Основной класс `CbrCurrencyRates` отвечает за получение данных о курсах валют и возвращает их в JSON-формате.

Для расширения функциональности были разработаны декораторы:

- `YamlCurrencyRatesDecorator` — преобразует данные о курсах валют в YAML;
- `CsvCurrencyRatesDecorator` — преобразует данные о курсах валют в CSV.

При этом исходный класс `CbrCurrencyRates` не изменяется. Декораторы принимают его объект внутрь себя и добавляют новое поведение.

Пример использования:

```python
rates = CbrCurrencyRates()

json_result = rates.operation()
yaml_result = YamlCurrencyRatesDecorator(rates).operation()
csv_result = CsvCurrencyRatesDecorator(rates).operation()
```

Такой подход показывает основную идею паттерна: объект можно «обернуть» в другой объект, чтобы получить дополнительную функциональность.

## Сохранение данных

В проекте реализовано сохранение результата в файлы разных форматов:

- `rates.json`;
- `rates.yaml`;
- `rates.csv`.

Каждый класс имеет метод `save`, который сохраняет результат работы в соответствующий файл.

## Проверка работы

Для проверки корректности программы были написаны тесты с использованием модуля `unittest`.

В тестах проверяется:

- получение данных в JSON-формате;
- преобразование данных в YAML;
- преобразование данных в CSV;
- сохранение результата в файлы;
- работа компонентов без зависимости от интернета за счет заранее подготовленных тестовых данных.

Запуск тестов:

```bash
python -m unittest test_currency_rates.py
```
